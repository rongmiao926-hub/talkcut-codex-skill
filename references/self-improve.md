# 规则更新与自进化

当用户说“记住这个问题”“以后按这个规则来”“把这个教训沉淀下来”时，不要只回文字说明，要把规则真正写回 skill 文件。

## 更新目标

- 总流程变化：更新 `SKILL.md` 或对应 workflow 文档
- 口误识别规则：更新 `references/user-habits/` 下合适的文件
- 字幕常见术语：更新 `references/subtitle-dictionary.txt`
- 通用字幕校对经验：更新 `references/subtitle-workflow.md`

## 更新原则

- 先回溯刚才真实发生的问题，再落规则
- 不要只在文档末尾追加“教训”，要把规则整合到最合适的章节
- 如果是新类别问题，就新建一个清晰的规则文件
- 修改后要告诉用户：改了哪条规则、写进了哪个文件

## 反模式

- 只说“我记住了”，但没有改文件
- 把具体规则堆到文末日志里，导致下次依旧找不到
- 没读现有结构就直接覆盖原规则
